## 2025-05-22 - Shared spaCy Model for Memory Efficiency
**Learning:** In a modular NLP pipeline, different components (like Cleaner, EntityExtractor, and TonalityAnalyzer) might depend on the same underlying linguistic model (e.g., spaCy's `en_core_web_sm`). Loading this model independently in each class constructor significantly increases the application's memory footprint and slows down the initial warm-up phase, as the same heavy model is deserialized multiple times.
**Action:** Identify shared model dependencies early. Initialize the model once at the coordinator level and inject it into the dependent components. This approach mirrors the existing optimization for SentenceTransformers in this codebase and should be applied to any heavy, read-only model shared across modules.
